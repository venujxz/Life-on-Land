Technical Description

Splash Screen (index.html)

  • Uses a <meta http-equiv="refresh"> tag to automatically redirect users to the home page after a short delay, simulating a splash/landing experience.
  • Animated CSS loader implemented with @keyframes for a smooth, modern loading effect.
  • Floating logo uses CSS animation and box-shadow for visual engagement.
  • Team member cards are laid out using CSS Flexbox for responsive alignment and spacing.
  • Semantic HTML5 structure (<header>, <main>, <footer>) for accessibility and SEO.
  • Accessible skip link for keyboard users to jump directly to main content.
  • All images include descriptive alt text for screen readers.
  • Responsive design with @media queries for mobile and desktop compatibility.

Volunteer Page (volunteer.html)

  • Volunteer opportunities are displayed as horizontally scrollable cards using CSS Flexbox and overflow-x: auto for mobile-friendly navigation.
  • Each card includes an image, title, description, and a "Join Now" button that triggers a modal popup with dynamic content.
  • Popup modal is implemented with JavaScript and ARIA roles for accessibility, including keyboard navigation and focus trap.
  • Modal form uses <label> elements for all inputs, and validates required fields before submission.
  • Emoji-based rating system for user feedback, with live character count and a 200-character limit enforced via maxlength and JavaScript.
  • Feedback form provides real-time validation, confirmation messages, and resets on cancel or submit.
  • All interactive elements are accessible by keyboard and provide visible focus styles.
  • Responsive design ensures usability on all devices, with modal and cards adapting to screen size.
  • Scroll lock is applied to the body when the modal is open to prevent background scrolling.
  • ARIA attributes and roles are used for improved screen reader support.

Content Page (content_ST1.html)

  • Uses internal anchor links and section IDs for easy navigation within the page.
  • Content is organized into semantic sections with clear headings (<h2>, <h3>).
  • Images include descriptive alt text and are styled with CSS for visual consistency.
  • Sticky "Go to Top" button implemented with position: fixed for improved usability.
  • Consistent color palette and spacing achieved using CSS variables and utility classes.
  • Responsive layout adapts to different screen sizes using @media queries.
  • All navigation and interactive elements are accessible by keyboard and screen readers.
  • Page structure and content validated using W3C HTML and CSS validators.

Challenges & Lessons Learned

One challenge was debugging responsive Flexbox layouts and ensuring animations worked smoothly across browsers. I learned to use browser dev tools and test on multiple devices.

Merging code with teammates sometimes caused version conflicts. This taught me to communicate more and commit changes carefully.

I also improved my understanding of semantic HTML and accessibility, especially for forms and navigation.

Integrating dynamic JavaScript features, such as modals and emoji-based ratings, required careful event handling and accessibility considerations. I learned to use ARIA roles and keyboard navigation to make interactive elements usable for all.

Ensuring consistent styling across all browsers was a challenge, especially with custom components like popups and feedback forms. I learned to use CSS resets and test on different platforms.

Managing and validating user input, such as character limits and required fields, helped me understand the importance of real-time feedback and error handling in web forms.

Adhering to Janet regulations and web accessibility standards made me more aware of legal and ethical responsibilities in web development.

Finally, balancing visual design with accessibility (such as color contrast and focus indicators) taught me to prioritize both aesthetics and usability for all users.

Accessibility Highlights

  • All images include descriptive alt text for screen readers.
  • Semantic HTML elements (header, nav, main, section, footer) used throughout.
  • Color palette chosen for sufficient contrast (meets WCAG guidelines).
  • Keyboard navigation supported with logical tab order and visible focus styles.
  • Form elements have associated label tags for accessibility.
